BENCHMARK SPECIFICATIONS

1. RECEIPT OVERHEAD (1-receipt-overhead.mjs)
   Target: <1% overhead
   Measures: createReceipt() time vs bare function
   Config: 10,000 iterations, 1,000 warmup
   Output: Median time, P95, P99, overhead %, phase breakdown

2. DELTA COMPRESSION (2-delta-compression.mjs)
   Target: <10% of original state size
   Measures: Delta proposal size vs full RDF state
   Config: 10,000 quads, 10 delta operations
   Output: Compression ratio, bytes per operation, efficiency gain

3. QUERY PERFORMANCE (3-query-performance.mjs)
   Target: <5% overhead
   Measures: SPARQL query time with/without receipts
   Config: 1,000 entities, 100 queries per type
   Types: Simple, filtered, join, aggregation, complex
   Output: Overhead per type, average, max

4. MEMORY USAGE (4-memory-usage.mjs)
   Target: <2% overhead
   Measures: Receipt chain memory overhead
   Config: 100 quads, 100 receipts, requires --expose-gc
   Output: Memory delta, overhead %, bytes per quad

5. COMPOSITION LATENCY (5-composition-latency.mjs)
   Target: <10% slowdown vs single module
   Measures: Multi-module composition time
   Patterns: Single, two-hop, three-hop composition
   Output: Latency per hop, overhead %
