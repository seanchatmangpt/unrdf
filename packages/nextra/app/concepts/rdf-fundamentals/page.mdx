# RDF Fundamentals

Learn the core concepts of RDF (Resource Description Framework) and how UNRDF implements them.

---

## What is RDF?

RDF is a standard for representing information as **graphs** of **linked data**.

**Key idea**: Everything is described as **triples** (subject-predicate-object).

```
Alice knows Bob
 ↓     ↓     ↓
Subject Predicate Object
```

---

## RDF Terms

RDF has four types of terms:

### 1. Named Nodes (IRIs)

Globally unique identifiers:

```javascript
const alice = dataFactory.namedNode('http://example.org/alice');
```

**Properties**:
- Must be valid IRIs
- Globally unique
- Dereferenceable (can be looked up)

---

### 2. Literals

Data values with optional types/languages:

```javascript
// Plain literal
const name = dataFactory.literal('Alice');

// Language-tagged
const greeting = dataFactory.literal('Hello', 'en');

// Typed literal
const age = dataFactory.literal('30', namedNode('http://www.w3.org/2001/XMLSchema#integer'));
```

**Properties**:
- Represent data values
- Can have language tags (`@en`, `@fr`)
- Can have datatypes (integers, dates, etc.)

---

### 3. Blank Nodes

Anonymous resources:

```javascript
const person = dataFactory.blankNode();
```

**Properties**:
- No global identifier
- Scoped to document
- Useful for complex values

---

### 4. Graphs (Named Graphs)

Group quads into datasets:

```javascript
const graph = dataFactory.namedNode('http://example.org/graph1');
```

---

## Triples vs Quads

### Triple

`(Subject, Predicate, Object)`

```javascript
const triple = dataFactory.quad(
  alice,  // subject
  knows,  // predicate
  bob     // object
);
```

### Quad

`(Subject, Predicate, Object, Graph)`

```javascript
const quad = dataFactory.quad(
  alice,
  knows,
  bob,
  graph  // optional graph
);
```

**UNRDF uses quads** to support named graphs (datasets).

## Vocabularies

RDF vocabularies define predicates and classes:

### Common Vocabularies

**FOAF** (Friend of a Friend):
```javascript
const foaf = (term) => namedNode(`http://xmlns.com/foaf/0.1/${term}`);

foaf('knows')  // http://xmlns.com/foaf/0.1/knows
foaf('name')   // http://xmlns.com/foaf/0.1/name
```

**Schema.org**:
```javascript
const schema = (term) => namedNode(`http://schema.org/${term}`);

schema('Person')  // http://schema.org/Person
schema('name')    // http://schema.org/name
```

**RDF/RDFS**:
```javascript
const rdf = (term) => namedNode(`http://www.w3.org/1999/02/22-rdf-syntax-ns#${term}`);
const rdfs = (term) => namedNode(`http://www.w3.org/2000/01/rdf-schema#${term}`);

rdf('type')        // rdf:type
rdfs('label')      // rdfs:label
```

## Knowledge Graphs

A knowledge graph is a collection of RDF quads:

```javascript
const kg = createStore();

// Add facts
kg.add(quad(alice, foaf('name'), literal('Alice')));
kg.add(quad(alice, foaf('knows'), bob));
kg.add(quad(bob, foaf('name'), literal('Bob')));
kg.add(quad(alice, foaf('age'), literal(30)));
```

**Graph visualization**:

```
alice --foaf:name--> "Alice"
  |
  +----foaf:knows--> bob --foaf:name--> "Bob"
  |
  +----foaf:age----> 30
```

---

## Pattern Matching

Query graphs using patterns:

```javascript
// Find all quads where Alice is the subject
kg.match(alice, null, null);

// Find all names
kg.match(null, foaf('name'), null);

// Find everything
kg.match(null, null, null);
```

**Pattern syntax**: `(subject?, predicate?, object?, graph?)`
- `null` = wildcard (matches anything)

## SPARQL

SPARQL is SQL for RDF:

```sparql
SELECT ?person ?name
WHERE {
  ?person foaf:knows bob .
  ?person foaf:name ?name .
}
```

In UNRDF:

```javascript
const results = store.query(`
  SELECT ?person ?name
  WHERE {
    ?person <http://xmlns.com/foaf/0.1/knows> <http://example.org/bob> .
    ?person <http://xmlns.com/foaf/0.1/name> ?name .
  }
`);
```

## RDF Serializations

RDF can be serialized in multiple formats:

### Turtle (.ttl)

```turtle
@prefix foaf: <http://xmlns.com/foaf/0.1/> .
@prefix ex: <http://example.org/> .

ex:alice foaf:name "Alice" ;
         foaf:knows ex:bob .
```

### N-Triples (.nt)

```ntriples
<http://example.org/alice> <http://xmlns.com/foaf/0.1/name> "Alice" .
<http://example.org/alice> <http://xmlns.com/foaf/0.1/knows> <http://example.org/bob> .
```

### JSON-LD (.jsonld)

```json
{
  "@context": { "foaf": "http://xmlns.com/foaf/0.1/" },
  "@id": "http://example.org/alice",
  "foaf:name": "Alice",
  "foaf:knows": { "@id": "http://example.org/bob" }
}
```

## Best Practices

### 1. Use Vocabularies

Reuse existing vocabularies instead of creating custom predicates:

```javascript
// ✅ Good
quad(alice, foaf('name'), literal('Alice'))

// ❌ Bad (custom predicate)
quad(alice, namedNode('http://mysite.com/name'), literal('Alice'))
```

### 2. IRI Design

Use hierarchical, dereferenceable IRIs:

```javascript
// ✅ Good
namedNode('http://example.org/people/alice')

// ❌ Bad (not dereferenceable)
namedNode('alice')
```

### 3. Literal Datatypes

Always specify datatypes for non-string literals:

```javascript
// ✅ Good
literal('30', namedNode('http://www.w3.org/2001/XMLSchema#integer'))

// ⚠️ OK but less precise
literal(30)
```
