# OStar-Unrdf Integration: Gap Closure Documentation

## 1. Overview: The Design-Runtime Unified Architecture
To ensure artifacts generated by `ostar` are fully compatible with the hardened `@unrdf` (L4/L5) runtime, we are formalizing the "Bridge" protocol. This protocol mandates that all generated code artifacts automatically include infrastructure-level constraints for causality, validation, and auditability.

## 2. Infrastructure Injection Requirements
All `ostar` templates (especially `mcp-tool`, `powl`, and `shacl`) must be updated to inject the following requirements during the generation phase:

### A. Causal Tracking (Vector Clocks)
- **Requirement**: Every generated handler must instantiate a `VectorClock` tied to the execution context.
- **Implementation**:
  ```javascript
  // Generated Injection:
  import { VectorClock } from '@unrdf/kgc-4d';
  // Inside handler(input, context):
  const vc = new VectorClock(context.nodeId || 'default');
  ```

### B. Constitutional Validation (SHACL/Receipts)
- **Requirement**: Any generated logic that executes bytecode must pass through the `HardenedAtomVM` facade to ensure cryptographic receipt validation.
- **Implementation**:
  - Automatically wrap generated tool logic in a `HardenedAtomVM.execute()` call if the artifact type is `powl` or `mcp-tool`.

### C. ACID Persistence (Atomic Append)
- **Requirement**: All state mutations must be abstracted through the `KGCStore.appendEvent` transactional barrier.
- **Implementation**: 
  - Templates generating SPARQL `INSERT`/`DELETE` must be deprecated in favor of `delta`-based event logging to ensure rehydration stability.

## 3. Template Update Protocol
The following templates in `ostar/templates/` are scheduled for refactoring to incorporate the Bridge:

1.  **`projected-artifact.njk` (mcp-tool)**: Inject `VectorClock` and `HardenedAtomVM` wrappers.
2.  **`projected-artifact.njk` (powl)**: Map `powl:Process` steps to `OP_SPAWN_SEQ` opcodes.
3.  **`projected-artifact.njk` (sparql)**: Refactor from raw `INSERT/DELETE` to `EventLog` delta objects.

## 4. Verification Suite
All artifacts generated using the new Bridge protocol must pass the `UNRDF_COMPLIANCE_CHECK`:
```bash

```
