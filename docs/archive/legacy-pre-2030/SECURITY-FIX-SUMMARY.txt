SECURITY FEATURES IMPLEMENTED
───────────────────────────────────────────────────────────────────────────────

1. INPUT VALIDATION (Zod schemas):
   - URISchema: Validates all RDF URIs (max 2048 chars, regex, pollution checks)
   - PathSchema: Validates all HTTP paths (max 512 chars, traversal/pollution checks)
   - LiteralSchema: Validates RDF literals (max 4096 chars, XSS checks)
   - HTTPMethodSchema: Validates HTTP methods (GET/POST/PUT/DELETE/PATCH/HEAD/OPTIONS)
   - CustomerIdSchema: Validates customer IDs (alphanumeric only)

2. HANDLER SANDBOXING (HandlerSandbox class):
   - validateHandler(): Static analysis blocks dangerous patterns
   - executeRestricted(): Runtime isolation, no process/require/global access
   - Error sanitization: No stack traces leak

3. AUTHENTICATION (AuthenticationManager class):
   - generateToken(): Crypto-secure 32-byte tokens
   - validateToken(): Token validation with 24hr expiry
   - hasRole(): RBAC role checking
   - revokeToken(): Token revocation support

4. XSS PROTECTION:
   - sanitizeForJSON(): Escapes <>"'/ characters
   - sanitizeResponse(): Deep object sanitization
   - Security headers: Content-Type, X-Content-Type-Options

5. MEMORY PROTECTION (Bounded RDFStore):
   - Configurable triple limit (default 10,000)
   - cleanup(): LRU-style cleanup
   - size(): Store size tracking
